## Step 1 — Sheet loader ตัวเดียว + header-driven ทุกตาราง

**บล็อกเกอร์ของทุกอย่าง**

### 🔴 สถานะปัจจุบัน (ต้องเข้าใจก่อนแก้)
โค้ดตอนนี้อ่านชีต **ทุกตัวผ่าน publish CSV URL** — ทั้ง BotLibrary (`SHEET_STEP_URL` / `SHEET_FAQ_URL` / `SHEET_CONFIG_URL` / `SHEET_FOLLOW_URL`) **และ Orders** (`SHEET_ORDERS_ID` ปัจจุบันเก็บ **CSV URL** ไว้สำหรับ *อ่าน*)
ส่วนการ *เขียน* Orders ใช้ service account (editor) แยกต่างหาก — อันนี้ถูกแล้ว คงไว้

**การรื้อครั้งนี้ครอบคลุมทั้ง Orders และ BotLibrary** — เปลี่ยนวิธี *อ่าน* จาก `fetch(CSV URL)` เป็น Google Sheets API ทั้งระบบ ไม่ใช่แค่ BotLibrary

### สิ่งที่ต้องทำ
- อ่านทุกแท็บผ่าน **Google Sheets API + service account เดิม** (ตัวที่เขียน Orders อยู่แล้ว) → ชีตไม่ต้อง publish สาธารณะ (เลขพร้อมเพย์/สูตรการขายอยู่ในนั้น)
- env ฝั่งชีตเหลือ **2 ตัว เป็น ID ล้วนทั้งคู่:**
  - **`SHEET_BOTLIB_ID`** (ใหม่) — แทน 4 ตัว `SHEET_*_URL` ของ BotLibrary
  - **`SHEET_ORDERS_ID`** — 🔴 **เปลี่ยนค่าจาก CSV URL → ID ล้วน** (อ่านและเขียนด้วยตัวเดียว)
- BotLibrary 8 แท็บ: `CSV_Step` `CSV_Objections` `CSV_Examples` `CSV_FAQ` `CSV_Follow` `CSV_Config` `CSV_Products` `CSV_Promo`
- util กลาง `resolveColumns(headerRow, spec)` → map ชื่อ header → index · **ใช้กับทุกแท็บ รวม Orders**
- 🔴 **Orders สำคัญที่สุด** — ปัจจุบัน `appendOrderRow` / `listPendingOrders` / `markOrderSent` ใช้ index คงที่ A–R → แทรกคอลัมน์ทีเดียวเขียนผิดช่องทั้งแถว
- **โหลดขนาน** (`Promise.all`) · **cache ก้อนเดียว** (Config 60 วิ · ที่เหลือ 5 นาที) · โหลดไม่ได้ → ใช้ cache เก่า
- header ไม่เจอ → **ปิดฟีเจอร์ + log error** (all-or-nothing) ห้าม fallback เป็น index เงียบๆ
- ⚠️ **ค้นวิธีอ่าน/เขียน Google Sheets API ปัจจุบันก่อนเขียนโค้ด** อย่าอ้างจากความจำ

### 🔴 ลำดับ ENV (สำคัญ — ผิดลำดับ = บอทล่มระหว่างทาง)
ENV กับโค้ดต้องเปลี่ยน **พร้อมกัน** ตราบใดที่โค้ดยังอ่าน CSV URL → ENV ต้องเป็น CSV URL
1. **ก่อนเริ่ม:** เพิ่ม `SHEET_BOTLIB_ID` (ID ล้วน) ได้เลย — โค้ดเก่ายังไม่เรียก ไม่กระทบ · **ห้ามแตะ `SHEET_ORDERS_ID`** (ยังต้องเป็น CSV URL จนโค้ดใหม่เสร็จ)
2. **หลังแก้โค้ด Step 1 เสร็จ:** เปลี่ยน `SHEET_ORDERS_ID` เป็น ID ล้วน + ลบ 4 URL ของ BotLibrary → **redeploy** (ไม่ redeploy = ค่าใหม่ไม่มีผล) → เทส
3. **Claude Code: จบ Step 1 ต้องบอกเจ้าของชัดว่าให้ตั้ง/ลบ/เปลี่ยน ENV ตัวไหนเป็นค่าอะไรบ้าง**

✅ **Acceptance:** สลับตำแหน่งคอลัมน์ในชีต Orders แล้วบอทยังเขียนถูกช่อง · อ่าน Orders + เขียน Orders ทำงานด้วย `SHEET_ORDERS_ID` (ID ล้วน) ตัวเดียว · 1 เทิร์นเรียก Google ไม่เกิน 1 ชุด

## Step 5 — Agent: Objections + Examples + กฎข้อ 10 ⭐ (หัวใจของการเทรน)

### 5.1 Objection matching (`lib/agent/objections.ts`)
1. keyword match ข้อความลูกค้า กับคอลัมน์ `ลูกค้าพูดแบบไหนบ้าง`
2. เจอ → ยัด **เต็มแถว** สูงสุด 2 อัน (`จำนวนข้อโต้แย้งที่ยัดเข้า prompt`)
3. **ไม่เจอ → ไม่ยัด** ห้ามยัดมั่ว
4. ยัด **สารบัญ** (id + ชื่อ) ทั้ง 17 แถวเสมอ (สั้น ~300 tokens)

🔴 **ห้ามยัด CSV_Objections ทั้ง 17 แถวเต็มทุกเทิร์น** — prompt บวม ตอบช้า บอทลืมกฎเหล็ก

### 5.2 Examples injection
match จาก `step_id` ปัจจุบัน และ/หรือ `objection_id` ที่เจอ → ยัดสูงสุด 3 แถว

### 5.3 คำสั่งบังคับใน systemInstruction
```
ข้อโต้แย้ง: ใช้ "ความกังวลที่แท้จริง" + "หลักการตอบ" ประกอบคำตอบให้เข้ากับสิ่งที่ลูกค้าพูด
จริงในเทิร์นนี้ · เคารพ "ห้ามทำ" เคร่งครัด · 🔴 ห้ามลอก "ตัวอย่างคำตอบที่ดี" คำต่อคำ ·
ตอบเสร็จวกกลับประตูการขายเดิมเสมอ

ตัวอย่างน้ำเสียง: ใช้เป็นแนวจังหวะและโทน ห้ามลอกถ้อยคำ
```
ถ้าไม่ห้ามลอก → ลูกค้า 2 คนพูดคนละสำนวนจะได้คำตอบเดียวกันเป๊ะ = **จับได้ทันทีว่าเป็นบอท**

### 5.4 กฎเหล็กข้อ 10
> **ไม่มีข้อมูล = บอกตรงๆ ว่าไม่มี แล้วเรียกคน**
> ห้ามเดา ห้ามตอบกว้างๆ ห้ามเปลี่ยนเรื่อง ห้ามใช้ความรู้ทั่วไปนอกชีต → `handoff=true`

### 5.5 Self-improving loop (ฟรี ทำเลย)
เพิ่ม `objection_detected` ใน `responseSchema` → log คู่กับ `objection_matched_by_code`
AI ตอบ `OBJ_PRICE` แต่ code ไม่ match = **สำนวนนั้นยังไม่อยู่ในชีต** → เจ้าของเอาไปเติมช่อง `ลูกค้าพูดแบบไหนบ้าง` ของแถวเดิม

✅ **Acceptance:** "ราคานี้ซื้อข้าวได้ 3 มื้อเลยนะ" (ไม่มีในตัวอย่างไหน) → บอทยังใช้หลักการ OBJ_PRICE ตอบได้ · คำตอบ**ไม่เหมือน**ช่อง "ตัวอย่างคำตอบที่ดี" คำต่อคำ · `stage` ยังอยู่ S2

## Step 9 — Regression harness 🔴 (จำเป็น ไม่ใช่ของแถม)

บอทที่ "เข้าใจ" จะตอบไม่เหมือนเดิมทุกครั้ง → **เทียบข้อความเป๊ะๆ ใช้ไม่ได้**
ต้องเทสด้วย **behavioral assertion**

สคริปต์ replay บทสนทนาเข้า handler ตรงๆ (mock LINE / Sheets / Blob) รันด้วย 1 คำสั่ง

### Assertion กลาง (ทุก scenario ต้องผ่าน)
- ❌ ไม่มีตัวเลขราคาที่ไม่มีอยู่ใน CSV_Promo/CSV_Products
- ❌ ไม่มีคำใน `คำต้องห้าม_โฆษณา`
- ❌ ไม่มีข้อความที่ลอก `ตัวอย่างคำตอบที่ดี` เกิน 80% (similarity)
- ✅ บอลลูนสุดท้ายเป็น**ข้อความ** เสมอ (ไม่ใช่รูป)
- ✅ `stage` อยู่ใน enum
- ✅ ไม่มี `{ตัวแปร}` หลุดไปหาลูกค้า
